簡介

可以直接看最後面的murmur…其實就是一個安裝過程紀錄，並且把安裝過程分享給大家。因為本身是物理出生而不是CS，但是有部份對於CS的需求，因此只好自己摸索一些東西並且記錄下來。會選用Hexo+github的原因是原本用在Blogger實在是讓人不爽惹，為什麼不能直接遠端上傳MarkDown文件阿！我大部分所需要的文件就只需要markdown就可以了，但是Blogger弄了很多複雜的東西有點令人感到不爽（當然是很方便拉），也因此在因緣際會底下就開始使用了這個台製的Hexo架構，試用了兩個月發現其實還不賴！就算是整個站掛點還是可以很快速的救回來（大概花了一半時間在打文件、一半時間在重新整理。

常用外掛、主題

由於Hexo是一個掛在npm底下的一隻程式（應該是可以這樣理解），也因此有很多大神幫忙製作了許多好用的插件（外掛）在Hexo上，有需要的話就參考看看吧，這是個人常用的一些外掛。

外掛安裝方式請在自己的blog子資料夾中開始以下的動作。
npm install [外掛名稱]
下面這個加了-g的是全域安裝，簡單講照著做就對了（誤）。簡單講，沒有global的就是只有在目標資料夾底下安裝所需的package。
npm install -g hexagon-cli

外掛

大部分的外掛都可以在官方外掛頁面上查詢到。

  • hexo-hexagon
    一目了然讓你知道你的Hexo是否變成了超級大怪物塞了一堆阿里佈達的東西。個人覺得他的安裝功能不是非常的好用…而且沒有unintall，個人基本上就是拿來查詢安裝了什麼外掛用的。
  • hexo-browsersync
    好用！這幾乎必備阿！前期在調校Blog的時候需要用到這個，如果說想要一邊編輯文章一邊看編輯之後的效果，也可以利用這個，簡單講，就是強大。詳細的安裝可以參考MoxFive的文章
  • hexo-deployer-git
    為了守護github，讓你的資料能夠正常的放到github上，這個東西「必裝」。除了安裝這個外掛外，在Github上也需要增加SSH Key才能讓通訊協定達成。有需要可以看github-ssh說明文件，或是參考下面Github的部份

順利上傳網誌至Github

  1. 看你的電腦裡面有沒有鑰匙
    ls -al ~/.ssh
    如果有如同下列這些的檔案，那就可以跳到步驟3.

    id_dsa.pub
    id_ecdsa.pub
    id_ed25519.pub
    id_rsa.pub

  2. 安裝ssh-keygen
    sudo dnf install ssh-keygen
  3. 在終端上產生新的key
    ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
  4. 將ssh鑰匙加到ssh-agent
    背景執行ssh-agent
    eval "$(ssh-agent -s)"
    將私人SSH鑰匙加入ssh-agent，注意檔名。
    ssh-add ~/.ssh/id_rsa
  5. 複製貼上
    超強複製貼上神器（其實我第一次見到它是在github的說明文件…，不過蠻好用的）
    sudo dnf install xclip
    複製你的鑰匙吧，貼到Settings -> SSH and GPG keys -> New SSH key 上面
    xclip -sel clip < ~/.ssh/id_rsa.pub
  6. check
    ssh -T git@github.com
    基本上順著這個流程走應該是沒問題，個人是沒有碰到什麼奇怪的事情就是XD

npm 砍掉重練

參考[Stack-Overflow][SOF-npmremove]
基本上就是把node_modules整個砍掉就是了，好孩子沒事不要學。

npm ls -gp --depth=0 | awk -F/ '/node_modules/ && !/\/npm$/ {print $NF}' | xargs npm -g rm
